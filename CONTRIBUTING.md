## How to Contribute

### Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub with:

1. A clear and descriptive title
2. A detailed description of the issue or suggestion
3. Steps to reproduce (for bugs)
4. Expected behavior vs actual behavior
5. Any relevant error messages (sanitized - no secrets!)

### Submitting Pull Requests

1. Fork the repository
2. Create a new branch for your feature or fix (`git checkout -b feature/my-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit your changes with a clear commit message
6. Push to your fork
7. Submit a pull request

### Pull Request Guidelines

- Follow the existing code style and conventions
- Update documentation if needed
- Keep changes focused and atomic
- Write clear commit messages
- Reference any related issues in your PR description

### Code Style

- Use consistent formatting
- Add comments for complex logic
- Keep scripts modular and reusable
- Follow PowerShell best practices for scripts

## Development Setup

1. Clone the repository
2. Copy `http-client.env.json.sample` to `http-client.env.json`
3. Configure your Azure AD B2C / Entra External ID credentials
4. Test changes locally before submitting

## Questions?

Feel free to open an issue for any questions about contributing.
